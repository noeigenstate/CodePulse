# 架构现状与改进

## 数据流（当前）

```text
CLI Hook ──POST /api/events──► StatusHub.ingest
磁盘 SessionSync ──稳态/watch──► StatusHub.ingest
QuotaRefreshWatcher ──轮询──► StatusHub.ingest
        │
        ├─ emit('event')  → SQLite persistEvent（同步，主线程）
        └─ emit('status') → 托盘 diff + IPC codepulse:status → Zustand UI
```

桌面端**不**使用 `/ws` 轮询；渲染层纯 IPC 推送。

## 关键组件

| 组件 | 路径 | 职责 |
|------|------|------|
| SessionSync | `packages/local-server/src/session-sync.ts` | 主动扫 Codex/Grok/Claude 磁盘 |
| QuotaWatcher | `packages/local-server/src/quota-watcher.ts` | 绑定 rollout 重读额度 |
| StatusHub | `packages/core/src/hub/index.ts` | 状态机 + 快照 |
| Hooks | `packages/hooks` | CLI 实时事件 |
| Desktop main | `apps/desktop/src/main` | IPC、托盘、SQLite、更新 |

## 现存问题（审查时点）

### 延迟

- 稳态扫盘 8s 过长
- watch debounce 600ms 偏保守
- 每次 Codex 扫盘可能全量 tasklist（Windows）

### 流畅

- 每个 ingest 立即全量 status IPC
- 托盘 key 含 token/lastEventAt → 工具调用频繁重建菜单
- 每事件同步写 SQLite

### 准确

- Hook 过期 limits **删除** vs Server **soft-reset 0%** 不一致
- 过期窗口 merge 允许高 % 盖掉 soft-reset 0%
