## v1.2.1

### 更新内容

#### 主动同步本机 CLI 会话

- 打开 CodePulse 后主动扫描本机会话（无需先发一条对话才出数据）：启动首扫、前几秒补扫、稳态轮询，窗口聚焦 / 系统唤醒时再扫。
- Codex：仅在本机有 `codex` 进程时同步；以近期仍活跃的 rollout 为准，避免把「从没从该项目打开 CLI」的沉寂项目拉进面板。
- Grok：仅同步 `active_sessions.json` 中且进程仍存活的会话，并合并 billing 周额度。
- 等额度重置时 CLI 长时间不写盘：放宽活跃窗口，并对绑定 rollout 持续重读额度；重置后先显示 0% / 可刷新，不再卡在「等待命令行同步额度」。

#### 本地隐私与统计

- 事件落库不再保存 raw / 完整 command；仅保留白名单扩展名等元数据与短预览。
- 启动时迁移清理旧库中的敏感列；统计页自愈缺失 `file_type_hints` 列，避免「统计聚合失败」。
- 大库分批清理与 WAL 繁忙容错，降低迁移中断导致残留 raw 的概率。

#### 本机安全加固

- 回环 API 增加本机共享密钥（`~/.codepulse/local-auth`）；Hook 自动携带，健康检查除外。
- 本地服务端口绑定失败时**不再写入** Claude / Codex / Grok Hook，避免事件打到其它占用端口的进程。
- 应用内更新：默认只从官方 GitHub 下载；HTTPS 与域名白名单（含重定向）；支持校验安装包 SHA-256。国内可按需设置 `CODEPULSE_UPDATE_ALLOW_MIRRORS=1` 启用镜像（官方地址仍优先）。

#### 其它

- 渲染进程启动时主动触发一次会话同步；统计后台与实时面板链路更稳。
- 补充 session-sync、额度刷新、本地认证与隐私迁移相关测试。
