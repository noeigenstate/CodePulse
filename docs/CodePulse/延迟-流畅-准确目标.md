# 延迟 · 流畅 · 准确目标

## 1. 同步延迟更低

| 路径             | 优化前                | 目标                 |
| ---------------- | --------------------- | -------------------- |
| Hook 成功        | 几十 ms               | 保持                 |
| 磁盘 watch       | debounce 600ms        | **150–250ms**        |
| 磁盘稳态         | 最长 ~8s              | **~3–4s**            |
| Windows 进程探测 | 每次扫盘 tasklist ≤3s | **缓存 3s，超时 1s** |

## 2. 使用更流畅

| 场景         | 优化前                         | 目标                                       |
| ------------ | ------------------------------ | ------------------------------------------ |
| 工具风暴     | 每事件全量 IPC + 托盘 + SQLite | status **32ms 合并**；托盘只 diff 语义字段 |
| 窗口空闲可见 | 多组 500ms 时钟                | 第二波：统一 now                           |

## 3. 信息更准确

| 问题                              | 目标                                         |
| --------------------------------- | -------------------------------------------- |
| Hook 删掉过期 limits → 粘住旧高 % | soft-reset **0% + 保留 resetsAt**            |
| 过期窗口 last-write 回写 88%      | **同 past resetsAt 取 min used%**（0% 粘住） |

## 4. 第三波工程目标

| 场景                        | 目标                           | 结果                     |
| --------------------------- | ------------------------------ | ------------------------ |
| 单一来源文件变化            | 不扫描另外两种 CLI             | dirtySources 定向扫描 ✅ |
| 窗口聚焦风暴                | 同一动作只发起一轮补扫         | 180ms 共享防抖任务 ✅    |
| symlink / junction 项目路径 | 同一真实目录只保留一个项目键   | Hook + 扫盘 realpath ✅  |
| 多项目面板                  | DOM 数量不随全部项目数线性增长 | >8 项窗口化渲染 ✅       |
