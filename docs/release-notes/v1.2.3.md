## v1.2.3

### 更新内容

#### 空闲项目 5 分钟后剔除

- 空闲 / 已完成项目约 5 分钟后从面板移除（额度条可保留）。
- 去掉 session-sync 的 keep-alive 刷新，避免卡片永远不算空闲。
- 仅账号额度变化不再重置项目活跃时间；Codex 近期写入窗口与 5 分钟空闲策略对齐。

#### 项目路径显示

- 用户主目录 / Desktop 等通用路径不再吞掉其下真实项目（避免 MetalMax 显示成 Administrator）。
- 无工作区路径的额度壳 / 探针事件**不再**画成「未识别项目」卡片；有额度时只保留额度条。

#### Claude 额度主动同步

- statusline：stdin `rate_limits` → OAuth usage → 本地缓存 `~/.codepulse/claude-quota.json` 三级补齐。
- session-sync 打开 Claude 时合并账号额度；无项目时也可显示额度壳。
- 支持 `utilization`、ISO `resets_at`、opus/sonnet 分模型周窗口。
- 收到有效额度后落盘缓存，便于下次启动主动回填。

#### 额度刷新时间

- 丢弃离谱的远未来 `resets_at`（如调试占位 `2000000000` 导致「刷新 2498 天」）。
- 展示层对超过约 10 天的倒计时显示「刷新 —」；合并时可用合理时间覆盖错误占位。

#### 其它

- 补充 Claude 额度缓存、空闲剔除、路径聚合与展示相关测试。
