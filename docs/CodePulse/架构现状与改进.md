# 架构现状与改进

## 数据流（当前）

```text
CLI Hook ──POST /api/events──► StatusHub.ingest
磁盘 SessionSync ──稳态全扫 / watch 按源脏扫──► realpath ──► StatusHub.ingest
QuotaRefreshWatcher ──轮询──► StatusHub.ingest
        │
        ├─ emit('event')  → SQLite persistEvent（同步，主线程）
        └─ emit('status') → 托盘 diff + IPC codepulse:status → Zustand UI
```

桌面端**不**使用 `/ws` 轮询；渲染层纯 IPC 推送。

## 关键组件

| 组件         | 路径                                         | 职责                          |
| ------------ | -------------------------------------------- | ----------------------------- |
| SessionSync  | `packages/local-server/src/session-sync.ts`  | 主动扫 Codex/Grok/Claude 磁盘 |
| QuotaWatcher | `packages/local-server/src/quota-watcher.ts` | 绑定 rollout 重读额度         |
| StatusHub    | `packages/core/src/hub/index.ts`             | 状态机 + 快照                 |
| Hooks        | `packages/hooks`                             | CLI 实时事件                  |
| Desktop main | `apps/desktop/src/main`                      | IPC、托盘、SQLite、更新       |

## 现存问题（审查时点 → 已处理 / 遗留）

### 延迟（第一波已处理）

- ~~稳态扫盘 8s 过长~~ → 3.5s
- ~~watch debounce 600ms 偏保守~~ → 200ms
- ~~每次 Codex 扫盘可能全量 tasklist~~ → 3s 缓存

### 流畅（第一/二波）

- ~~每个 ingest 立即全量 status IPC~~ → 32ms coalesce
- ~~托盘 key 含 token/lastEventAt~~ → 语义 diff
- ~~每事件同步写 SQLite（扫盘）~~ → 跳过 internal sessionSync/quotaRefresh
- ~~多 useNow 独立 timer~~ → 共享 clock 1s
- ~~App 整树订阅 store~~ → slice selectors
- ~~任一 watch 触发三来源全扫~~ → dirtySources 按源扫描
- ~~focus/bootstrap 重复全扫~~ → 180ms 共用防抖任务
- ~~多项目一次挂载全部卡片~~ → >8 项启用动态行高虚拟列表
- 遗留：tool 风暴 lifecycle 仍同步写 SQLite

### 准确（第一/二波）

- ~~Hook 过期 delete vs soft-reset~~ → 对齐 0%
- ~~过期 min %~~ → 禁止旧高回写
- ~~Codex fallback 绕过 soft-reset~~ → 上下文-only fallback
- ~~共享额度清空 quotaBuckets~~ → 按 family 保留
- ~~Claude 周额度 first-hit~~ → 模型族优先
- ~~statusline 一律 exact~~ → 官方 used_percentage 才 exact
- ~~symlink/junction 路径形成重复项目~~ → Hook + 扫盘入口 realpath
