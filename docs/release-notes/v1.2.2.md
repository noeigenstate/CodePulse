## v1.2.2

### 更新内容

#### Codex 周额度稳定

- 修复多项目 / 多 rollout 并存时，每周额度在正确值与过期快照（如 2%）之间乱跳。
- 账号额度按「同计费窗口取最高 used%」共享到各项目，不再被最新 mtime 的陈旧会话或 Spark 桶抢占。
- StatusHub 对同一 `resetsAt` 窗口的用量只升不降；过期 soft-reset 仍可归零。
- 额度条展示与后台轮询优先权威快照，避免活跃但过期的会话盖住正确周用量。

#### Hook 路径更稳

- Claude / Codex / Grok 的 Hook 改为指向用户主目录稳定入口 `~/.codepulse/hooks/bin/*`。
- 真实安装目录写在 `~/.codepulse/hook-runtime.json`，每次打开 App 自动刷新；换盘或重装后无需手改 CLI 配置。
- Windows 不再把认证 token 塞进易碎的 `cmd /c` 嵌套引号，避免 statusline / hook 跑不起来。

#### Claude 额度同步

- statusline 显式保留并转发 `rate_limits`，合并上下文补丁时不再丢掉周/5h 额度。
- 适配 Claude 仅下发 `seven_day_opus` / `seven_day_sonnet` 等分模型周窗口的情况。
- 等待额度同步时，Claude 分屏同时显示 5h 与每周占位条，布局更稳。

#### 其它

- 本地认证优先使用当前 local-server 实际在用的 token 配置 Hook。
- 补充 session-sync、额度合并与稳定 Hook 路径相关测试。
