## v1.1.0

### 更新内容

#### 本地开发数据统计后台

- 右上角「后台」进入全屏统计台：总览本机 AI 编程的 Token 消耗、开发时长、项目与对话。
- 支持今日 / 近 7 天 / 近 30 天切换，趋势图可按日 / 周 / 月查看；一键刷新从本机 SQLite 同步最新记录。
- 提供模型使用占比、高峰时段热力图、项目消耗排行、使用洞察、项目/文件类型分布与本地效率评分。
- 数据仅存本机、不上传云端；会话会关联工作区路径，便于按项目汇总。

#### Grok 上下文同步

- 修复 Grok 任务执行中有时不显示上下文窗口：活动会话从 `updates.jsonl` 读取实时 `params._meta.totalTokens`，窗口大小来自 `models_cache.json`。
- `signals.json` 仍作为任务结束后的优先来源；忽略 `turn_completed.usage` 累计用量，避免误当上下文。

#### 额度与稳定性

- 修复 Codex 在 fork / resume 后额度来源错乱：更准确识别当前会话 transcript，避免误用父会话或 Spark 额度。
- 合并更早的 `token_count` 中的 rate_limits，减少上下文-only 刷新覆盖周额度信息的情况。
- 继续收敛 Spark 额度条误显：按当前活跃或最近使用的模型显示对应周额度。

#### 应用内更新

- 发现新版本时展示 Release 更新说明（What's new），便于升级前快速了解变更。
